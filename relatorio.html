Cupom-coletor

Neste problema, o colecionador de cupons tem como objetivo completar o seu ‚Äú√°lbum‚Äù. No contexto, o colecionador realiza sorteios com reposi√ß√£o dos itens do √°lbum at√© completar a sua cole√ß√£o. A pegunta para este problema √© ‚Äúespera-se quantos sorteios com reposi√ß√£o para completar todos os itens da cole√ß√£o?‚Äù

Simula√ß√£o: Cupom-coletor

Considere um √°lbum com \(N\) itens distintos. Em cada passo, um itens ser√° sorteado do conjunto, sendo que cada item tem probabilidade \(1/N\) de ser escolhido. O item √© reposto e o sorteio √© feito novamente. O processo continua at√© que todos os \(N\) itens √© observado pelo menos uma vez, ou seja, o ‚Äú√°lbum √© completado‚Äù. A vari√°vel de interesse √© o n√∫mero de sorteios necess√°rios \(T_n\).

CupCol <- function(N){
  aux <- c() #vetor auxiliar para armazenar os itens j√° sorteados
  Tn <- 0 #para contar o n√∫mero de sorteios realizados
  
  while(length(aux) < N){ #loop at√© completar o √°lbum 
    Tn <- Tn + 1 #contagem dos sorteios
    i <- sample(1:N, 1) #sorteio de um item com probabilidades iguais
    
    if(i %in% aux){ #verifica se o item j√° foi observado
      next #pula para pr√≥xima itera√ß√£o
    }
    else{ #se o item n√£o foi observado ainda
      aux <- c(aux, i) # adiciona o item no vetor aux
    }
  }
  return(Tn) #retorna o numero de sorteios feitos 
}

replicador <- function(M, N){
  aux <- c()
  for(i in 1:M){ #faz M iteracoes do processo
    x <- CupCol(N) #chama a fun√ß√£o 
    aux <- c(aux, x) #armazena o valor x em um vetor
  }
  return(aux)#retorna o vetor com os M numeros de sorteios 
  
}

Teoria: Cupom-coletor

O valor esperado do \(T\) (n√∫mero de sorteios) √© (resultados em(1)):

\[ E(T) = nH(n) \]

onde o \(H(n)\) √© o n-√©simo n√∫mero harm√¥nico:

\[ H(n) = 1 + \frac{1}{2}+...+\frac{1}{n-1}+\frac{1}{n} \]

Para valores grandes de \(n\), pode ser aproximado como:

\[ E(T) = nH(n) \approx n\ln{(n)} +\gamma n +\frac{1}{2} +O\bigg(\frac{1}{n}\bigg) \]

onde \(\gamma \approx 0.577215665\) √© a constante Euler-Mascheroni e \(O\bigg(\frac{1}{n}\bigg)\) representa um termo de erro que decresce proporcionalmente a \(1/n\).

A vari√¢ncia de \(T\) √© (resultados em (1)):

\[ Var(T) < n^2\bigg(\frac{\pi^2}{6}\bigg) \]

Assim, podemos encontrar o valor esperado e vari√¢ncia aproximados para cada n:

#fun√ßoes para calcular a medias e variancias teoricas
media <- function(n){
  return(n*log(n) + 0.577215665*n + 1/2 )
}
variancia <- function(n){
  return((n^2)*(pi^2)/6)
}

Parad√≥xo do anivers√°rio

Neste problema, em grupo de \(k\) indiv√≠duos aleat√≥rios, queremos encontrar a probabilidade de duas pessoas ter a mesma data de anivers√°rio.

Simula√ß√£o: Parad√≥xo do anivers√°rio

Fixe \(N = 365\). Para cada \(k \in [2, 60]\) pessoas ser√° feito um sorteio com reposi√ß√£o de 1 at√© \(N\), \(k\) vezes. Se tiver um item j√° observado, o sorteio conta como sucesso. Se n√£o, fracasso. A vari√°vel de interesse √© um indicador booleano se foi sucesso ou fracasso no sorteio.

ParAni <- function(N, k){
  aux <- c()#vetor auxiliar para armazenar as datas sorteadas
  bool <- FALSE# booleano para ver se foi sucesso ou fracasso
  for(i in 1:k){ #loop para sortear as datas k vezes
    x <- sample(1:N, 1) #sorteio de uma data, de 1 at√© N=365
    if(x %in% aux){ #verifica se houve datas repetidas(colis√£o)
      bool <- TRUE #booleano torna true=sucesso
      break #termina as itera√ßoes
    }
    else{
      aux <- c(aux, x)#se n√£o, adiciona a data no vetor aux
    }
    
  }
  return(bool)#retorna o booleano
}

repetidor <- function(k, M){
 aux <- list() #lista para armazenar os vetores
for(i in 2:k){ #para cada k, sera feito os loops
  vetor <- c() #vetor para armazenar os booleanos
  for(l in 1:M){ 
   x <- ParAni(N = 365, i) #sorteios 
   vetor <- c(vetor, x) #adiciona o novo bool no vetor
  }
  aux <- c(aux, list(vetor)) #adiciona o vetor na lsta
  }
return(aux)#retorna a lista
 }

Teoria: Paradoxo do Anivers√°rio

Para calcular a probabilidade de \(n\) pessoas em uma sala , em que pelo menos duas pessoas tenham o mesmo aniver√°rio, calculamos a probabilidade do complementar, ou seja, a probabilidade de que \(n\) pessoas n√£o tenham o mesmo anivers√°rio(referencias em 2):

\(\bar{p}(n) = 1\cdot \bigg(1 - \frac{1}{365}\bigg)\cdot \bigg(1 - \frac{2}{365}\bigg)\cdot\cdot\cdot \bigg(1 - \frac{n-1}{365}\bigg) = \frac{365!}{365^n (365-n)!}\)

Com isso, encontramos a probabilidade desejada, de \(n\) pessoas, pelo menos duas pessoas terem o mesmo anivers√°rio:

\(p(n) = 1 - \bar{p}(n)\)

Fazemos ent√£o o c√≥digo para encontrar as probabilidades:

f_prob_an <- function(n){
  if(n > 365){
    return(1) #se n maior que 365, a probabilidade √© 1
  }
  else if (n < 2){
    return(0) #se n menor que 2,  a probabilidade √© 0
  }
  else {
    prob_diferentes <- 1
    for (i in 1:(n-1)) {
      prob_diferentes <- prob_diferentes * (365 - i) / 365
    }
    # Probabilidade de pelo menos 2 terem o mesmo anivers√°rio
    return(1 - prob_diferentes)
  }
}
k = 60
vetor_aniv <- c()
for(i in 2:k){
  y <- f_prob_an(i)
  vetor_aniv <- c(vetor_aniv, y)
}

Conclus√µes

Percebemos no projeto que, simulando um problema em algoritmo e obter uma grande quantidade de amostras independentes, podemos encontrar m√©dias, vari√¢ncias ou probabilidades emp√≠ricas que tem boa precis√£o. Esse m√©todo √© chamado de M√©todo do Monte Carlo, que explica que ao obter amostras aleat√≥rias de grande n√∫mero, este tende ao par√¢metro desejado, ou seja:

\[ \hat{\theta_n} = \frac{1}{n}\sum{}g(X_i) \to \theta \]

quase certamente, pela Lei dos Grandes n√∫meros.

Assim, conclu√≠mos que √© poss√≠vel obter respostas de problemas probabil√≠sticos analicamente, ao simular o problema em c√≥digos e obter v√°rias amostras aleat√≥rias, conforme foi feito no projeto.
